# Schemas

All schemas are pydantic v2 BaseModels and live in `ac_harness/schemas.py`. Every model carries `schema_version: int = 1`. Unknown fields are tolerated (`extra="allow"`) so AC-Core schema drift doesn't break the harness.

## Candidate (§5.1)

An architecture under consideration. May come from AC-Core, a manual config, or another search system.

```text
id, source ∈ {ac_core, manual, external},
architecture_config_path?, baseline_id?,
changed_fields, candidate_metadata,
ac_core_prediction_id?,
provenance, created_at
```

## ACCorePrediction (§5.2)

Compile-time prediction from AC-Core. **Imported, never generated by the harness.**

```text
id, candidate_id, hardware_id, runtime, workload_id,
predicted_metrics, predicted_bottlenecks,
risk_label?, compiler_notes?,
provenance, created_at
```

## ExperimentPlan (§5.3)

What the planner says to measure next.

```text
id, name, experiment_type ∈ {
  kernel_microbench, serving_bench, small_training_ablation,
  quality_eval, import_external_result
},
candidate_ids, decision_unblocked, uncertainty_target,
estimated_cost, required_resources, config,
status ∈ {planned, approved, running, completed, failed, skipped},
rationale?, provenance, created_at
```

## Measurement (§5.4)

One row per observed metric value. Curves are many rows.

```text
id, candidate_id, experiment_id?,
measurement_type ∈ {kernel, serving, training, eval, manual_import},
metric_name, metric_value, metric_unit?,
hardware_id?, runtime?, workload_id?,
step?, seed?, extra, provenance, created_at
```

## FittedCalibration (§5.5)

A ridge / residual / interaction fit.

```text
id, name, target ∈ {throughput, serving, quality_residual, interaction},
input_measurement_query, n_measurements,
functional_form, coefficients, uncertainty,
holdout_error?, valid_for, created_at,
status ∈ {draft, validated, stale, insufficient_data},
needed_measurements (only meaningful for insufficient_data),
provenance
```

## DecisionState (§5.6)

Central user-facing summary record.

```text
id, name, candidate_ids, observed_summary,
current_frontier, uncertainty_summary,
disagreements, recommended_next_experiments,
human_decisions_required, notes?,
created_at, provenance
```

## HumanDecision (§14)

Surfaced at the human-review checkpoints listed in `decision_state.md`.

```text
id, decision_type, prompt, options,
selected_option?, rationale?,
created_at, resolved_at?, provenance
```

## Provenance

Every inserted record carries a `provenance` dict with at minimum:

- `created_at` (ISO timestamp)
- `host`, `python_version`
- `git_commit` (if available)
- `source_path` (if imported)
- `command` (the function or CLI call that produced it)

Provenance is non-optional in spirit: `EvidenceStore` auto-fills any insert that arrives without it.
